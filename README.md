## 📖 アプリ概要

**RT91 Navi** は、スター・ウォーズに登場するアストロメク・ドロイド（R2-D2 のような）をモチーフにした、ブラウザで動く会話型ウィジェットアプリです。

画面上に 3D で表示されたドロイドに話しかけると、天気・時刻・日付を教えてくれます。スター・ウォーズに関する 150 以上の百科事典エントリも内蔵しています。Google の Gemini AI を連携すると、より自由な会話も楽しめます。

---

## ✨ 主な機能

| 機能 | 説明 |
|------|------|
| ☀️ 天気案内 | 現在地の天気・気温・湿度・風速を表示。APIキー不要で無料 |
| ⏰ 時刻表示 | 「今何時？」と話しかけると現在時刻を教えてくれる |
| 📅 日付表示 | 「今日は何日？」で今日の日付・曜日を回答 |
| 🌟 スター・ウォーズ百科事典 | 150 以上のキャラクター・惑星・メカ・名言などの情報を内蔵 |
| 🤖 Gemini AI連携 | Google の AI（Gemini 2.5 Flash Lite）と接続してより自由な会話が可能 |
| 🌌 3D ビジュアル | Three.js で描かれた宇宙空間・地球・ドロイドのリアルタイム3D描画 |
| ☁️ 動的気象雲システム | 世界30地点の実際の気象データから4種類の雲をリアルタイム生成 |
| 📱 PWA対応 | スマホのホーム画面に追加するとアプリのように使える |
| 🔌 オフライン対応 | 一度開いたらオフラインでもある程度動く（Service Worker） |

## 🚀 使い方

### 1. 基本の会話（APIキー不要）

画面右下のチャット欄にメッセージを入力して ▶ ボタンか Enter を押すだけです。

```
「天気を教えて」     → 現在地の天気を表示
「今何時？」         → 現在時刻を表示
「今日は何日？」     → 今日の日付・曜日を表示
「こんにちは」       → 挨拶を返してくれる
「できること」       → 機能一覧を表示
「ダース・ベイダー」 → スター・ウォーズの百科事典から情報を表示
「ライトセーバー」   → ライトセーバーの種類や色の解説
「ヨーダ」           → キャラクター情報を表示
```

> 🌟 スター・ウォーズに関する質問は 150 以上のキーワードに対応しています（キャラクター、惑星、メカ、組織、名言など）。

> 💡 位置情報の許可を求められた場合は「許可」を選ぶと、現在地の天気が取得できます。拒否しても東京の天気が表示されます。

### 2. Gemini AI との自由会話（任意）

右上の ⚙ ボタンから **Gemini API キー** を設定すると、設定したキーワード以外の質問にも自然に回答できるようになります。モデルは **Gemini 2.5 Flash Lite** を使用し、RT91 ドロイドとしてのキャラクターを維持したまま、会話の文脈を保持した自然な対話が可能です（直近20ターン分の会話履歴を保持）。

**APIキーの取得方法（無料）：**
1. [Google AI Studio](https://aistudio.google.com/app/apikey) にアクセス
2. Google アカウントでログイン
3. 「APIキーを作成」ボタンを押してコピー
4. アプリの ⚙ 画面に貼り付けて「保存して閉じる」

> 🔒 APIキーはあなたのデバイスのみに保存されます（`localStorage` を使用）。GitHubや外部サーバーには一切送信されません。

### 3. スマホのホーム画面に追加（PWA）

- **iOS Safari**: 「共有」→「ホーム画面に追加」
- **Android Chrome**: メニュー（⋮）→「アプリをインストール」または「ホーム画面に追加」

ホーム画面からアプリのように全画面で起動できます。

### 4. 3D ドロイドの操作

| 操作 | 動作 |
|------|------|
| ドラッグ | カメラを回転 |
| スクロール | ズームイン・アウト |

ドロイドは自律的にウロウロ移動し、話しかけるとカメラを向いて頭を振ります。

## 🛠️ 使用技術の解説（初学者向け）

### Three.js（スリーJS）— 3D描画ライブラリ
> [cdnjs.cloudflare.com から読み込んでいます（r128）]

ブラウザ上で 3D グラフィックを描くためのライブラリです。通常、ブラウザで 3D を描くには `WebGL` という低レベルの API を直接書く必要がありますが、Three.js はそれを使いやすくラップしてくれています。

- **Scene（シーン）**: 3D 空間全体。ドロイド・地球・星空がすべてここに配置されています
- **Camera（カメラ）**: 視点（40° FOV）。ドラッグ操作で動かすのはこのカメラです
- **Mesh（メッシュ）**: 形（ジオメトリ）＋質感（マテリアル）の組み合わせ。ドロイドの各パーツはすべて Mesh です
- **Light（ライト）**: 4 点照明システム（キー・フィル・リム・ボトム）。ACES フィルミック トーンマッピングで映画的な質感を表現
- **Shadow（シャドウ）**: PCFSoftShadowMap（2048×2048）によるリアルなソフトシャドウ
- **requestAnimationFrame**: ブラウザに「次の描画タイミングでこの関数を呼んで」と伝える仕組み。これをループさせることでアニメーションを実現しています

### Open-Meteo API — 天気情報＋雲データ（無料・APIキー不要）
> [api.open-meteo.com](https://open-meteo.com)

API（Application Programming Interface）とは、外部サービスのデータをプログラムから取得する仕組みです。Open-Meteo はオープンソースの気象データ API で、登録不要・完全無料で使えます。

このアプリでは Open-Meteo API を **2 つの目的**で使っています：

1. **チャットの天気案内**: GPS（`navigator.geolocation`）で現在地を取得し、気温・体感温度・湿度・風速・天気コード・最高/最低気温を受け取ってカード形式で表示（30 分キャッシュ）
2. **3D 地球の雲レンダリング**: 世界 30 地点の気象データ（WMO 天気コード・雲量）を取得し、天気コードに応じた雲の種類（層雲・積雲・サイクロン・巻雲）と雲量に応じたサイズを動的に決定して地球上に配置

### Gemini API — AI会話
> [Google AI Studio](https://aistudio.google.com)

Google が提供する生成 AI の API です。テキストを送ると AI が返答を生成します。このアプリでは **Gemini 2.5 Flash Lite** モデルを使用し、RT91 ドロイドのキャラクターとして振る舞うように「システム指示」を与えています。直近 20 ターン分（40 メッセージ）の会話履歴を保持することで文脈を踏まえた自然な対話を実現しています。

### PWA（Progressive Web App）
> Service Worker + Web App Manifest

ウェブサイトをスマホアプリのように使えるようにする技術の総称です。

| 技術 | 役割 |
|------|------|
| `manifest.json` | アプリ名・アイコン・テーマカラーなどを定義。ホーム画面追加を実現 |
| `sw.js`（Service Worker） | バックグラウンドで動くスクリプト。ファイルをキャッシュしてオフラインでも動くようにする |

### localStorage — データの保存
ブラウザが提供するデータ保存機能で、サーバーなしにデータを端末に保存できます。このアプリでは Gemini API キーの保存に使っています。ブラウザを閉じても消えません。

### Canvas — 3D描画の土台・テクスチャ生成
HTML の `<canvas>` 要素は「自由に絵が描けるキャンバス」です。Three.js はこの Canvas を使って WebGL で 3D 描画を行っています。このアプリでは Canvas を 3D 描画の出力先としてだけでなく、**テクスチャの生成にも活用**しています。JavaScript の Canvas 2D API（グラデーション・図形描画）で地球の陸地テクスチャや地面の大理石タイルテクスチャを動的に生成し、`THREE.CanvasTexture` を通じて 3D マテリアルに適用しています。外部画像ファイルなしに複雑な質感を実現するのが特徴です。

### Visual Viewport API — iOS キーボード対応
iOS Safari ではソフトキーボードを開くとビューポートのサイズが変わらないため、キーボードが入力欄を隠してしまう問題があります。`window.visualViewport` を使うとキーボードの表示状態を検知でき、チャットパネルの位置を動的にずらすことでこの問題を回避しています。

## 💎 こだわりポイント

### 🔴 忠実な RT91 ドロイドモデル
Three.js のプリミティブ（球・円柱・箱・トーラス）を組み合わせて、手動でコードだけで作成した 3D ドロイドモデルです。赤白配色のドーム（10 枚のパネル・赤白 8 分割レーダーディッシュ・メインアイ＋セカンダリアイ＋ホロプロジェクター）、ブルーストライプ付きのテーパーボディ（ベントグリル・ポート・パネルライン・ユーティリティアームドア）、油圧シリンダー＆ピストン付きの二脚＋埋め込みホイール付き中央脚のトリポッド構造など、約 500 行のコードで細部まで作り込んでいます。

### 🌍 手描きの 3D 地球
テクスチャ画像を一切使わず、JavaScript で 1024×512 の `<canvas>` に 50 以上の陸地ポリゴン（ユーラシア、アフリカ、南北アメリカ、オーストラリア、インドネシア諸島、ニュージーランドなど）を描いてテクスチャを生成しています。砂漠・ジャングル・山脈・サバンナなど地形ごとに色分けし、極域の氷冠も 6 段階のグラデーションで表現。大気圏の 2 層グロー効果（内側 + リム）・スペキュラーマップ（海面の反射）・地軸の傾き（23.4°）まで再現しています。

### ☁️ 4種類の物理ベース雲シミュレーション＋動的気象連携

地球の大気圏上に 4 種類の雲を物理法則に基づいて生成・配置しています。**Mie 散乱**近似として `roughness=1.0, metalness=0.0` の完全拡散マテリアルを使用し、雲は `earthMesh` の子要素として地球の自転に連動します。

| 雲タイプ | 特徴 |
|----------|------|
| **層雲（Stratus）** | 水平に広がる扁平な雲層。メインシート＋6パッチで層状構造を再現 |
| **積雲（Cumulus）** | 対流セルを 13 個の球体で積み上げた雲頂。高さとともに不透明度が下がる密度勾配付き。**凝結高度（LCL）** を扁平な楕円体で再現した平坦な雲底 |
| **熱帯低気圧（Cyclone）** | 目・壁雲・3 本のスパイラルレインバンド（各 8 セグメント）・上層アウトフローを再現。北半球は反時計回り、南半球は時計回りにコリオリ力を反映 |
| **巻雲（Cirrus）** | 高高度（8〜12 km）の氷晶。ジェット気流に沿う 5 本の薄い繊維で構成。透明度が高く繊細な表現 |

さらに、ページ読み込み時に **Open-Meteo API から世界 30 地点の実際の気象データを取得**し、WMO 天気コードに基づいて雲の種類・サイズ・配置を動的に決定します。API 取得失敗時には 23 個の静的雲定義がフォールバックとして描画されます。

### 🏛️ 神殿風大理石タイルの地面
512 × 512 の Canvas2D でプロシージャル生成した大理石テクスチャです。`createLinearGradient` で温かみのあるクリーム〜アイボリーのベース色を作り、半透明の細線を 18 本引いて大理石の脈模様を再現。4 × 4 のタイルグリッドを目地幅 3px で分割し、各タイルに個別の色ばらつき・ハイライト（上・左端）・シャドウ（下・右端）のベベル処理を施しています。交互タイルには控えめなダイヤモンド装飾も入っています。

さらに、経年劣化を表現する詳細なエフェクトも実装しています：
- **マイクロスクラッチ**（16 本）：短く細い表面の擦り傷
- **通常スクラッチ**（8 本）：中程度の長さの使用傷
- **深いスクラッチ**（4 本）：長く目立つ深い傷
- **ひび割れ**（3 系統）：分岐するフラクタル構造のクラック

バンプマップで目地の凹み・タイル面の微妙な膨らみ・傷の溝と隆起縁を立体的に表現しています。

### 💫 3,500 個の星空
ランダム分布の点群（BufferGeometry）で宇宙空間を表現。星の色に微妙な温度差（青白〜暖色）を持たせ、ゆっくり回転させることでリアルな宇宙感を演出しています。

### 🌟 スター・ウォーズ百科事典（150 以上のエントリ）
RT91 ドロイドの「ナビゲーター」としてのアイデンティティを活かし、スター・ウォーズ全般に関する 150 以上の百科事典エントリを内蔵しています。キャラクター（ダース・ベイダー、ヨーダ、ルーク等）、惑星（タトゥイーン、ホス、ダゴバ等）、組織（ジェダイ、シス、銀河帝国等）、テクノロジー（ライトセーバー全色、デス・スター、X-Wing等）、映画・シリーズ（エピソード1〜9、マンダロリアン、アソーカ等）、名言（12 以上の有名セリフ）を網羅しています。Gemini AI がなくてもキーワードで即座にアクセスできます。

### 🤖 自律的なドロイドの動作
ドロイドは 2〜6.5 秒ごとにランダムな目標地点へ移動します（ワンダリング）。移動方向にボディを向け、速度に応じて前傾きが変わり、足が交互に揺れます。ボディの微妙な横揺れやアイドル時の縦揺れも表現。話しかけるとカメラ方向を向いて頭を振る反応アニメーションも実装しています。カメラはドロイドの動きを滑らかに追従します。

### 📱 モバイルファースト設計
- 縦画面でもドロイド全体が収まるようカメラ距離をアスペクト比で自動調整
- チャット入力欄のフォントサイズを 16px 固定（iOS Safari の自動ズーム防止）
- Visual Viewport API でソフトキーボードが入力欄を隠さない制御
